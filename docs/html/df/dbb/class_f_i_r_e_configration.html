Detailed Description

Represents the root configuration document loaded from the FIRE configuration YAML file.

FIREConfigration is the primary configuration model for the FIRE application, serialized from and deserialized to YAML via YamlDotNet. It defines all the settings required to control the three-step file processing pipeline (collect, generate, execute).

Configuration Version: The configuration includes a ConfigurationVersion that must match SupportedConfigurationVersion (1.00). This ensures forward/backward compatibility and prevents accidental use of incompatible configuration files.

File Discovery and Storage: FilesRootPath lists the directory roots to scan for source files during the collect phase. DataBasePath and DataBaseFileName specify the location of the SQLite database file where records are persisted.

File Operations and Path Templates: Action specifies the file operation (Copy, Move, or Link). RootPath is the base directory for output files. SortingPatern and FileNamePatern are path templates containing placeholders like {Make}, {Model}, {DateTimeOriginal} that are resolved using collected metadata during the generate phase.

Text Substitution and Extension Overrides: StringReplacements defines global string replacements applied to all resolved paths. FileExtensions provides per-extension configuration overrides for specific file types, allowing different sorting patterns, keyword sets, and file operations for (e.g.) images vs. videos.

Deserialization and Normalization: The Parse and Load methods deserialize YAML using YamlDotNet. The Normalize method fills in defaults for all properties, ensuring predictable behavior even if some fields are omitted from the YAML file.

Loading and validating a configuration:

var config = FIREConfigration.Load("Configuration.yaml");
config.EnsureSupportedConfigurationVersion();
Console.WriteLine($"Root path: {config.RootPath}");
Console.WriteLine($"Sorting pattern: {config.SortingPatern}");
Console.WriteLine($"File extensions configured: {config.FileExtensions.Count}");

Dictionary<string, FileExtensionConfiguration> FIREConfigration::FileExtensions { get

Gets or sets a dictionary of file extension-specific configurations.

Keys are file extensions (e.g., ".jpg", ".mp4", ".raw") and values are FileExtensionConfiguration objects that override the root configuration for files with that extension. ◆ FileSorting

List<string> FIREConfigration::FileSorting { get

Gets or sets the ordered list of metadata keys used to sort records before path generation.

Each entry represents one sort criterion. Typical values are metadata keys such as Make, Model, or date-time component selectors like MetaCreationTime.Year, MetaCreationTime.Month, MetaCreationTime.Day.

The criteria are applied in the listed order (primary, secondary, tertiary, ...). If the list is empty, no explicit sorting is performed.
