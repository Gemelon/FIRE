Configuration Reference

(C) 2026 by Thomas Stoll

Core Principles

  • Configuration is YAML-based.
  • Placeholder resolution is metadata-driven.
  • Counter usage is explicit and opt-in.

Essential Keys

  • ConfigurationVersion
  • FilesRootPath
  • DataBasePath
  • DataBaseFileName
  • Action
  • MediaRootPath
  • SortingPatern
  • FileNamePatern
  • FileExtensions

Placeholder Rules

  • Use {Keyword} for direct values.
  • Use {Keyword.SubValue} for composed values (for example date parts).
  • Counter is active only when template contains {Counter:Dx} style placeholders.

Counter Persistence

For photo/video archives, counter values must remain stable. Used counter values are persisted so deleted numbers are not reused later.

Sidecar Files

Configure sidecar extensions under a primary extension entry. Sidecars inherit primary target naming logic and keep their own extension.

Keyword Resolution

  • Define ordered KeyWords lists.
  • ValAttribute: LOWEST chooses smallest/earliest value.
  • ValAttribute: HIGHEST chooses largest/latest value.

Best Practices

  • Start with a small input folder.
  • Run collect + generate before execute.
  • Use inspect/diagnostic commands for metadata validation.
  • Keep configuration version pinned to supported value.
