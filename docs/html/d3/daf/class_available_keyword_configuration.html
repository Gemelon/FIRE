Detailed Description

Represents keyword metadata configuration for use within a file extension context.

AvailableKeywordConfiguration defines how a specific keyword (metadata key) should be extracted and selected for use in path templates. It specifies the metadata source, data type, and value selection strategy (e.g., select the lowest, highest, or first value when multiple values are available).

Metadata Source: The Source field indicates where the metadata should be extracted:

  • FILEINFO: Extract from file system properties (size, creation time, etc.)
  • EXIFTOOL: Extract from embedded EXIF/IPTC/XMP metadata using ExifTool

Data Type and Value Selection: The DataType field indicates the expected data type (e.g., "STRING", "INT", "DATETIME"). The ValAttribute field specifies the selection strategy when multiple values are found:

  • LOWEST: Select the minimum value
  • HIGHEST: Select the maximum value
  • FIRST: Select the first value encountered

Keyword Names: The KeyWords list contains the actual metadata field names to query from the source. For EXIFTOOL, these are EXIF tag names (e.g., "Make", "Model", "DateTimeOriginal"). For FILEINFO, these are system property names (e.g., "CreationTime", "LastWriteTime"). When multiple keywords are listed, the first matching value is used.

Normalization: The Normalize method fills in defaults after deserialization to ensure consistent behavior even if properties are omitted from the YAML file.

Configuring a keyword to extract the camera make from EXIF:

var makeKeyword = new AvailableKeywordConfiguration
{
DataType = "STRING",
Source = "EXIFTOOL",
ValAttribute = "FIRST",
KeyWords = new List<string> { "Make" }
};

string AvailableKeywordConfiguration::DataType { get

Gets or sets the data type of the metadata value.

Valid values include "STRING", "INT", "INTEGER", "DATETIME", "DATE", "TIME", etc. This is used to interpret and parse the metadata value correctly during path template resolution and value selection. Defaults to "STRING" if not specified.

◆ Default

string AvailableKeywordConfiguration::Default { get

Gets or sets the fallback value used when no configured keyword is found in the selected source.

If specified, this value is persisted instead of NA when no configured keyword yields a value. For DataType=DATETIME, the value may be a date/time string (e.g. "2024-12-31 00:00:00") or the special token NOW for the current local date and time.
