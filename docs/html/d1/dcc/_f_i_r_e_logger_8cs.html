enum FIRELogLevel

Numeric severity levels for FIRE log entries.

The numeric values allow simple greater-than comparison for level filtering.

  • Debug (0) – Fine-grained diagnostic information.
  • Info (1) – Normal stage-start/complete messages.
  • Warning (2) – Non-fatal anomalies (e.g. missing metadata).
  • Error (3) – Recoverable errors that affected a single file.
  • Critical(4) – Unrecoverable failures that aborted a stage.

Thread-safe, per-stage Markdown log writer for the FIRE catalog.

File naming: Each stage gets its own daily file: {LogFileName}.col.{yyyyMMdd}.md, .gen., .exe.. When no stage is active (e.g. pre-stage warnings), the suffix .misc. is used.

Rotation: When the active file grows beyond LoggingConfiguration.MaxFileSizeBytes, writing continues in a new segment: {name}.col.{date}.1.md, .2.md, … The recommended size limit is 10 MB so any common text editor can open the file.

Retention: At construction time, files older than LoggingConfiguration.MaxAgeDays days are deleted from the log directory. A 30-day window covers a reasonable audit period; reduce it in high-volume environments to limit disk usage.

Format: Every entry is one Markdown table row: | timestamp | LEVEL | stage | message | Each new file starts with a Markdown table header.
