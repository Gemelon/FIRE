Overview

The solution contains three projects:

  • FIRE (class library): core API and domain logic
  • FIRE.Console (console app): test/quick-start client around API calls
  • FIRE.Tests (test project): automated validation for API behavior

API-Centric Layering

Core Layer (FIRE)

Responsibilities:

  • load and validate YAML configuration
  • collect metadata from multiple sources
  • persist pipeline data in SQLite
  • generate deterministic target paths
  • execute file operations
  • publish progress/state updates
  • provide localized API messages

Adapter Layer (FIRE.Console)

Responsibilities:

  • parse command line input
  • map commands to API calls
  • render user-facing CLI output
  • support diagnostics and quick validation

Non-responsibilities:

  • business logic ownership
  • long-term UI runtime model

Validation Layer (FIRE.Tests)

Responsibilities:

  • keep API behavior stable
  • protect pipeline invariants
  • support refactoring safety

Why API First

Prioritizing API completion delivers:

  • stable integration contracts
  • reusable logic for future UI/automation
  • reduced duplication across clients
  • clearer separation between domain and presentation
