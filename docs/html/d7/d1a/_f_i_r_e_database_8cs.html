class FIREFileMetaData

Represents a single metadata entry stored together with a database record.

Each FIREFileMetaData instance encapsulates a key-value pair extracted from a file, along with metadata about the value's type and source origin. Multiple metadata entries are grouped into a single FIREDbRecord.

This class is used as a domain model in the FIRE library. It is persisted to the SQLite database via Entity Framework Core and mapped to/from the internal FIREFileMetaDataEntity class.

Typical metadata keys include:

  • Make, Model (camera information from EXIF)
  • DateTimeOriginal, DateTimeDigitized (from EXIF)
  • CreationTime, ModificationTime (from file system)
  • Custom keyword names resolved from configuration

The DataSource value typically indicates which IMetadataSource provided the entry (e.g., "EXIFTOOL", "FILEINFO").

Creating and populating metadata entries:

var metadata = new FIREFileMetaData
{
Key = "Make",
Value = "Apple",
TypeName = typeof(string).FullName,
DataSource = "EXIFTOOL"
};

class FIREStatusRecord

Represents the current status and state of the database content and recent persistence operations.

There is exactly one FIREStatusRecord row in the database at all times (with Id equal to 1). It is created during FIREDatabase initialization and is updated after every persistence operation (Save, Add, Clear, etc.).

The status indicates the result of the most recent operation:

  • Initialized – Database was just created
  • Saved – Last operation completed successfully
  • Failed – The last operation encountered an error
  • Saving – Operation in progress (rarely observed)

The Valid field indicates whether the stored data set is consistent and complete. It is set to false when an error occurs during persistence. The ErrorMessage field captures the exception message if Valid is false.

Checking the database status after a batch operation:

var status = database.StatusRecord;
if (!status.Valid)
{
Console.WriteLine($"Error: {status.ErrorMessage}");
}
else
{
Console.WriteLine($"Last operation: {status.Status} at {status.TimeStamp:O}");
}

◆ FIREDbRecord

class FIREStatusRecord FIREDbRecord = []

Provides a list-like interface to FIREDbRecord file entries backed by a SQLite database.

FIREDatabase combines Entity Framework Core with SQLite persistence to provide a simple in-memory collection abstraction over stored file records. It implements the IList<T> interface, allowing standard collection operations while automatically persisting changes to the database.

Architecture: The class maintains an in-memory _records list and a FIREDatabaseContext for EF Core operations. When you modify the collection (Add, Remove, Clear, etc.), the changes are immediately persisted to the underlying SQLite database via PersistCurrentState, which wraps the entire operation in a transaction.

Initialization: When you create a new instance, the SQLite file is created if it doesn't exist. If recreateIfExists is true, any existing database is deleted first. A singleton status row is automatically initialized to track database state.

Reload and Persistence: The Reload method reloads all records from the database into memory. The Save method persists the current in-memory list to the database. Every modification operation (Add, Remove, etc.) automatically calls PersistCurrentState to flush changes to the database immediately.

Status Tracking: The StatusRecord property provides access to the singleton status row, which records the result and timestamp of the most recent operation. If a persistence operation fails, the status is set to "Failed" and the error message is recorded.

Thread Safety: This class is not inherently thread-safe. Multiple concurrent modifications to the same database from different threads or processes will likely result in conflicts. Ensure external synchronization if concurrent access is required.

Creating a database and adding records:

using var db = new FIREDatabase("D:\\data\\fire.db", recreateIfExists: true);
var record = new FIREDbRecord
{
SourceFilePath = @"D:\Photos\IMG_1234.jpg",
FileMetaDatas = new List<FIREFileMetaData>
{
new() { Key = "Make", Value = "Nikon" }
}
};
db.Add(record);
Console.WriteLine($"Total records: {db.Count}");
Console.WriteLine($"Status: {db.StatusRecord.Status}");

Represents one complete file record persisted in the FIRE database.

Entity Framework Core entity class mapping to the FIREStatusRecord table.

Entity Framework Core entity class mapping to the FIREDbRecords table.

Each FIREDbRecord corresponds to a single source file that has been scanned and catalogued by the FIRE pipeline. It stores file identification information, paths, and all metadata collected from the file.

A FIREDbRecord is the primary domain model for FIRE. It typically progresses through the three-step pipeline as follows:

  1. collect: SourceFilePath is set, and FileMetaDatas are populated from configured metadata sources.
  2. generate: TargetFilePath is computed by evaluating path templates with resolved metadata values.
  3. execute: The file operation (Copy, Move, or Link) is performed from SourceFilePath to TargetFilePath.

The VolumeSerialNumber and FileId are used on Windows/NTFS systems to detect hard links and duplicates, avoiding unnecessary processing of the same physical file.

Creating a record after file collection:

var record = new FIREDbRecord
{
SourceFilePath = @"D:\Photos\IMG_1234.jpg",
VolumeSerialNumber = volumeSerialNumber,
FileId = fileId,
FileMetaDatas = new List<FIREFileMetaData>
{
new() { Key = "Make", Value = "Apple", DataSource = "EXIFTOOL" },
new() { Key = "Model", Value = "iPhone 15 Pro", DataSource = "EXIFTOOL" }
}
};
