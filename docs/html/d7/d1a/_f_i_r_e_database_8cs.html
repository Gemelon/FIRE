class FIREFileMetaData

Represents a single metadata entry stored together with a database record.

Each FIREFileMetaData instance encapsulates a key-value pair extracted from a file, along with metadata about the value's type and source origin. Multiple metadata entries are grouped into a single FIREDbRecord.

This class is used as a domain model in the FIRE library. It is persisted to the SQLite database via Entity Framework Core and mapped to/from the internal FIREFileMetaDataEntity class.

Typical metadata keys include:

  • Make, Model (camera information from EXIF)
  • DateTimeOriginal, DateTimeDigitized (from EXIF)
  • CreationTime, ModificationTime (from file system)
  • Custom keyword names resolved from configuration

The DataSource value typically indicates which IMetadataSource provided the entry (e.g., "EXIFTOOL", "FILEINFO").

Creating and populating metadata entries:

var metadata = new FIREFileMetaData
{
Key = "Make",
Value = "Apple",
TypeName = typeof(string).FullName,
DataSource = "EXIFTOOL"
};

string DataSource { get Gets or sets the source of the metadata value.

Valid values include "EXIFTOOL" (extracted via ExifTool) and "FILEINFO" (extracted from the file system). This field is useful for auditing and debugging the metadata collection process.

long Id { get Gets or sets the database identifier (primary key) of the metadata row.

This is auto-generated by the SQLite database when the row is inserted. Typically, you do not set this value manually; it is managed by Entity Framework Core.

string Key { get Gets or sets the metadata key name.

Examples: "Make", "Model", "DateTimeOriginal", "CreationTime". This key corresponds to the keyword name defined in the FIRE configuration.

set
string TypeName { get Gets or sets the CLR type name of the logical value.

This is the fully qualified name of the original .NET type, such as "System.String", "System.Int32", "System.DateTime". It is recorded to support type-aware parsing and formatting when the value is used in template placeholders.

string Value { get Gets or sets the metadata value as a string.

All metadata values are stored as strings, regardless of their logical type. The logical type is recorded separately in the TypeName property. For example, an integer resolution value might be stored as "300" with TypeName set to "System.Int32".

class FIREFileMetaData RegularFile = 0

Categorizes files by their role in the collection.

This enum allows FIRE to distinguish between different file types during processing. Regular files are processed normally through the metadata extraction and path generation pipeline. Sidecar files inherit the target path from their associated primary file (only the extension differs).

This enumeration distinguishes between primary files that are processed through the normal metadata extraction pipeline and auxiliary sidecar files that inherit path generation from their associated primary file.

During the collection phase, sidecar files are automatically detected based on the SidecarFileExtensions configuration and added to the database with SidecarFile classification. In the generate phase, they inherit the target directory and base filename from their primary file. In the execute phase, they use the global action setting.

A regular primary file processed through the full FIRE pipeline. This is the default classification for all collected files.

Regular files (e.g., .jpg, .mp4, .dng) undergo metadata extraction, template-based path generation, and file operations according to their extension-specific configuration.
